How a new post is written

Every post in this feed follows the same shape. To add one, copy the prompt below into any AI chatbot, paste a short description of what changed, and the chatbot will return a JSON object ready to drop into public/announcements.json.

Entry shape 
{
  "id": "YYYY-MM-DD-short-kebab-slug",
  "author": "Waystar Education",
  "date": "YYYY-MM-DD",
  "commit": "",
  "title": "<one short English sentence>",
  "body": "<2 to 4 plain English sentences>"
}
Hard rules
  1. author is always the literal string "Waystar Education". Never change it.
  2. date is today's date in ISO YYYY-MM-DD. The page renders it as May 11, 2026.
  3. id is <date>-<kebab slug>, lowercase ASCII only.
  4. Leave commit as an empty string unless a short Git SHA is provided.
  5. Write title and body in English. title is one sentence; body is 2 to 4 sentences.
  6. The body covers (a) what changed in user-visible terms, and (b) how a regular site visitor benefits. No bullet lists, no links, no code.
  7. Forbidden characters in title and body: the em dash (—, U+2014) and the en dash used as a sentence break (–, U+2013). Use commas, periods, parentheses, or semicolons instead.
Copy this prompt into any AI chatbot 
You are writing one announcement entry for the EduFinder by Waystar website. Produce exactly one JSON object in the shape below, with no surrounding prose, ready to be appended to the "entries" array of public/announcements.json (the feed sorts latest first).

Shape:
{
  "id": "YYYY-MM-DD-short-kebab-slug",
  "author": "Waystar Education",
  "date": "YYYY-MM-DD",
  "commit": "",
  "title": "",
  "body": "<2 to 4 plain English sentences>"
}

Hard rules:
1. "author" is the literal string "Waystar Education". Never change it.
2. "date" is today in ISO YYYY-MM-DD format.
3. "id" is "-" using a to z, 0 to 9, and hyphens only.
4. Leave "commit" as "" unless I give you a short Git SHA, in which case use that string.
5. Write "title" and "body" in English. "title" is one sentence. "body" is 2 to 4 sentences.
6. "body" covers (a) what changed in user-visible terms and (b) how a regular site visitor benefits. No bullet lists, no links, no code, no marketing fluff.
7. Forbidden characters in "title" and "body": the em dash (the character U+2014) and the en dash (U+2013) used as a sentence break. Use commas, periods, parentheses, or semicolons instead.
8. Before you finish, scan your "title" and "body" for U+2014 and U+2013 and rewrite any sentence that contains either character. Treat their presence as a failed answer.
9. Output the JSON object only. No prose around it, no markdown code fence, no commentary.
